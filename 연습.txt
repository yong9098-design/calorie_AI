요청 1
- 로그인된 사용자가 새로고침하거나 재접속할 때 온보딩 `Step 1` 화면이 잠깐 보이는 문제를 없애줘.
- 현재 원인 후보:
  - `screen-onboarding`가 기본 `active` 상태로 먼저 렌더됨
  - 이후 `initApp()`에서 `/api/config` -> `getSession()` -> `loadProfile()` 후 홈으로 전환됨
- 원하는 방향:
  - 앱 부팅 중에는 중립적인 splash/loading 상태만 보이게 하고, 인증/세션 판별 후에만 실제 화면을 렌더
  - 인증 사용자의 경우 온보딩 플래시가 절대 보이지 않게 처리
  - 게스트/비로그인/온보딩 미완료 플로우는 유지
  - cold start나 첫 요청 지연이 있더라도 UI가 잘못된 화면을 먼저 보여주지 않게 처리

요청 2
- 기록 탭 식사 기록 로딩 체감을 개선해줘.
- 현재 원인 후보:
  - 기록 탭이 `preloadPromise`를 기다림
  - 캐시 없으면 날짜별로 다시 직접 쿼리
  - `select('*')`로 불필요한 컬럼까지 로딩
  - `image_url`에 base64 이미지가 들어가서 payload가 큼
  - `notes`도 리스트 조회에 불필요하게 같이 내려옴
  - 날짜 이동 시 디바운스 150ms
- 원하는 방향:
  - 기록 탭 진입 시 가능한 빨리 화면이 보이도록 구조 개선
  - `preloadPromise` 대기 때문에 기록 탭이 막히지 않게 조정
  - 리스트 조회는 필요한 컬럼만 가져오도록 최소화
  - 첫 렌더 체감을 개선하고, 네트워크/파싱 비용을 줄여줘
  - 기존 기능 회귀 없이 수정
- Supabase first-hit latency는 일부 있을 수 있으니, 그것을 고려하되 앱 구조상 병목도 같이 개선해줘.
