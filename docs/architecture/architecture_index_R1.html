  • Supabase: 인증, 식단 기록, 동기화

발표 포인트

사용자는 홈에서 촬영 또는 검색으로 진입하고, 분석 결과를 반드시 수정 가능한 상태로 확인한 뒤 저장합니다.

UX 원칙

  • 처음 사용자는 온보딩으로 목표를 설정
  • 재방문자는 홈으로 바로 복귀
  • AI 결과는 자동 저장 전 사용자가 검수
  • 기록과 통계가 다음 행동을 유도
Data Flow

사진 한 장이 식단 기록이 되는 과정

이미지는 브라우저에서 전처리되고, Edge API가 Gemini Vision과 Supabase 저장을 조율합니다. 사용자는 결과를 확인하고 분량을 조정할 수 있습니다.

품질 포인트

AI 분석은 추정값이므로 사용자가 수정할 수 있어야 합니다. 저장 전 검수 단계를 둔 것이 서비스 신뢰도를 높입니다.

데이터 안정성

  • 분석 중간 상태를 로컬 캐시에 보관
  • API 응답은 앱에서 쓰기 쉬운 형태로 정규화
  • Supabase 저장과 로컬 반영을 함께 처리
  • 네트워크 실패 시 재시도 여지를 남김

  • SPEC이 기능 범위와 성공 기준을 정의

  • Evaluator가 회귀, UI, 요구사항 적합성 확인
  • 통과 후 CHANGELOG로 이력화
