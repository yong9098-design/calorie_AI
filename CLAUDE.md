# 칼로리 트래커 하네스 오케스트레이터

> 사용자 요청 → 자동 설계 → 자동 구현 → 자동 검수 → PASS/FAIL 판정

---

## 📊 전체 워크플로우

```
┌─────────────────────────────────────────────────────────────┐
│  ① 사용자 요청                                              │
│     요구사항 작성 (docs/USER_REQUEST.md)                   │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│  ② PRD Builder Skill                                        │
│     요구사항 분석 → PRD 자동 생성 (docs/PRD.md)            │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│  ③ Planner Agent (agents/planner.md 참고)                 │
│     PRD 기반 → 상세 설계서 생성 (docs/SPEC.md)            │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│  ④ Generator Agent (agents/generator.md 참고)             │
│     SPEC 기반 → 앱 코드 생성 (output/index.html)          │
│     자체 검증 (docs/SELF_CHECK.md)                         │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│  ⑤ Evaluator Agent (agents/evaluator.md 참고)            │
│     SPEC vs Code 검수                                       │
│     4가지 평가 기준으로 채점 (evaluation_criteria.md)      │
│     결과 보고 (docs/QA_REPORT.md)                          │
└─────────────────────┬───────────────────────────────────────┘
                      │
           ┌──────────▼──────────┐
           │  합격 여부 판정     │
           └──────────┬──────────┘
                      │
           ┌──────────┴──────────┐
           │                     │
      ✅ PASS            ❌ FAIL/조건부
           │                     │
         완료              Generator로 피드백 반영
                           (최대 3회까지)
```

---

## 🎯 각 단계 역할

| 단계 | 이름 | 입력 | 출력 | 역할 |
|------|------|------|------|------|
| ② | PRD Builder | 요구사항 | PRD.md | 제품 정의 |
| ③ | Planner | PRD.md | SPEC.md | 화면 설계 |
| ④ | Generator | SPEC.md | index.html | 코드 구현 |
| ⑤ | Evaluator | SPEC.md + HTML | QA_REPORT.md | 품질 검증 |

---

## 🔄 반복 루프

- **Evaluator 판정이 FAIL/조건부** → Generator로 돌아가 피드백 반영
- **최대 반복**: 3회
- **3회 후에도 미합격**: 현재 상태로 완료 + 이슈 보고

## ⚙️ 에이전트 호출

각 에이전트의 상세 지시사항은 `agents/` 폴더에 있습니다:
- `agents/planner.md` — Planner 에이전트 지시
- `agents/generator.md` — Generator 에이전트 지시
- `agents/evaluator.md` — Evaluator 에이전트 지시
- `agents/evaluation_criteria.md` — 품질 평가 기준

**중요**: Generator와 Evaluator는 **반드시 다른 서브에이전트**로 호출하세요.
(= "만드는 AI와 평가하는 AI를 분리"하는 핵심)

## [필수] CHANGELOG 자동 업데이트 규칙

**모든 기능 구현이 완료될 때마다 반드시 `CHANGELOG.md`를 업데이트하라.**

해당 시점: Evaluator가 "합격" 판정을 내린 직후, 또는 사용자가 구현 완료를 확인한 시점.

### 작성 형식

```markdown
## [YYYY-MM-DD] <기능명 또는 작업 요약>

### 구현 내용
- 구체적으로 추가/변경된 사항 (기능 단위로 기술)

### 핵심 로직
- 주요 알고리즘, API 연동 방식, 아키텍처 결정 사항

### 변경된 파일
- `파일경로` — 변경 이유 한 줄 요약

### QA 결과
- 최종 점수: X.X/10 (반복 횟수: N회)
```

### 작성 원칙
- 전문적이고 기술적인 한국어 톤 사용
- 미래에 코드를 처음 보는 개발자가 흐름을 파악할 수 있는 수준으로 작성
- 파일이 루트에 없으면 아래 헤더로 신규 생성
- 파일이 없으면 아래 헤더로 신규 생성:
  ```markdown
  # Changelog

  AI 칼로리 트래커 프로젝트 변경 이력
  ```
