4️⃣ 데이터 흐름 상세: 음식 사진 분석 (Food Analysis Flow)
사용자가 음식 사진을 촬영하고 AI가 분석하며 DB에 저장되는 전체 과정 
sequenceDiagram
    participant User as 👤 사용자
    participant App as 📱 App
index.html
    participant Camera as 📷 Camera API
    participant ImageUtil as 🖼️ Image Utils
    participant EdgeFunc as 🔌 /api/analyze
    participant Gemini as 🤖 Google Gemini
    participant Supabase as 🗄️ Supabase
    participant LocalDB as 💾 localStorage

    User ->> App: 카메라 버튼 클릭
    App ->> Camera: 사진 촬영 요청
    Camera ->> User: 카메라 앱 오픈
    User ->> Camera: 음식 사진 촬영
    Camera -->> App: Image Blob

    App ->> ImageUtil: Blob → Base64
    ImageUtil -->> App: base64String
    App ->> LocalDB: 임시 저장

    Note over App,LocalDB: 요청 준비
    App ->> EdgeFunc: POST /api/analyze
{image_base64, model}

    Note over EdgeFunc,Gemini: 분석 실행
    EdgeFunc ->> Gemini: content-image-base64
+ ANALYSIS_PROMPT
    Gemini -->> EdgeFunc: {food_name, calories,
protein, carbs, fat,
quantity, components}

    Note over EdgeFunc,Supabase: DB 저장
    EdgeFunc ->> Supabase: INSERT meal_logs
(분석 결과 저장)
    Supabase -->> EdgeFunc: {id, created_at}

    EdgeFunc -->> App: {status: 'success',
meal_log_id}

    Note over App,LocalDB: UI 업데이트
    App ->> LocalDB: 결과 캐시
    App ->> App: 분석 결과 화면 표시
    App -->> User: ✅ 완료
(사용자 수정 가능)
🔄 흐름 요점
Base64 변환 후 전송 (CORS 우회)
Gemini 2.5 Flash/Pro 모델 선택 가능
결과는 JSON Schema로 검증됨
사용자가 수동으로 수정 가능
